# sync-rules

プロジェクトのコードベースを分析し、`.claude/rules/` にコーディング規約のルールファイルを自動生成する Claude Code プラグインです。

## 機能

- ソースコードのパターン、ディレクトリ構成、設定ファイルを自動分析
- トピック別のルールファイルを生成（コードスタイル、テスト、セキュリティ等）
- 既存の linter / formatter 設定を尊重（ESLint, Prettier, Biome 等と矛盾しない）
- 更新・同期モードでルールを最新の状態に維持
- モノレポ対応（path-scoped ルール）
- 言語非依存で、検出した言語に合わせた具体例を含む

## インストール

マーケットプレイスを追加し、プラグインをインストールします:

```
/plugin marketplace add almondoo/sync-rules
/plugin install sync-rules@sync-rules-marketplace
```

## 使い方

任意のプロジェクトでスキルを実行します:

```
/sync-rules:sync-rules
```

プラグインは以下の手順で動作します:

1. プロジェクトのファイル構成、設定ファイル、ソースコードをスキャン
2. コーディングパターン、命名規則、アーキテクチャを検出
3. 生成計画をユーザーに提示
4. ユーザーの承認後に `.claude/rules/` にルールファイルを生成

## 生成されるルールファイル

| ファイル | 生成条件 | 内容 |
|---------|---------|------|
| `code-style.md` | 常に生成 | インデント、行長、インポート順、命名規則、関数設計 |
| `testing.md` | テストファイルが存在 | テスト構造、命名、モック戦略 |
| `security.md` | Web/API プロジェクト | 入力バリデーション、認証・認可、秘密情報管理 |
| `api-design.md` | API 層が検出 | エンドポイント設計、リクエスト/レスポンス形式 |
| `error-handling.md` | ソースファイルが存在 | エラー型、ログレベル、リカバリパターン |
| `debugging.md` | ロギング/可観測性ツール検出 | ロギング規約、デバッグツール、可観測性 |
| `architecture.md` | アーキテクチャパターン検出 | 依存方向、レイヤー責務、禁止パターン |
| `workflows.md` | CI/CD 設定が検出 | ブランチ戦略、PR 規約、デプロイフロー |

## ローカル開発

プラグインをローカルでテストする場合:

```
claude --plugin-dir ./plugins/sync-rules
```
